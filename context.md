## What Gekko Is

Gekko is an autonomous **agent-to-agent payment marketplace** built for the **MetaMask Smart Accounts Kit × 1Shot API × Venice AI hackathon**. A user submits a research goal; four AI agents coordinate via ERC-7710 delegations, pay each other in USDC on Base Sepolia using direct ethers.js transfers, and deliver a full research report — all without human intervention.

**Stack**: Venice AI (private LLM inference) · x402 micropayment protocol · ERC-7710 delegation chain · ERC-7715 permission grants · ethers v6 USDC transfers · Base Sepolia testnet

## How the Data Flows

### Goal execution (`POST /api/goal`):

```
User → POST /api/goal (optional permissionContext from ERC-7715)
  → server.js (rate limiting, budget capping)
    → GekkoOrchestrator.executeGoal(goal, budget)
      1. _verifyBalance() — USDC balance via ethers on Base Sepolia (threshold: 0)
      2. _planSubtasks() — LLM or heuristic → 3 subtasks: research, validate, write
      3. For each subtask:
         a. registry.discover(capability) — find cheapest agent
         b. escrow.createEscrow() — in-memory session (no external service)
         c. agent.execute(task) — calls Venice AI via x402 proxy
         d. orchestrator.pay() — direct USDC transfer via ethers (simulated if unfunded)
      4. validator.validate() — fact-check via Venice deepseek-v3.2 or mistral-small
      5. writer.synthesize() — full report via Venice mistral-small
      6. Return { success, report, audit }
```

### Venice AI payment flow (x402):
```
agent.callAPI('venice', 'chat', body)
  → x402Client.fetch('/api/venice/chat', ...)
    → POST /api/venice/chat with no X-PAYMENT header
      → x402Middleware returns 402 + { paymentRequired: { recipient, amount, token } }
    → client signs EIP-191 message, retries with X-PAYMENT header
      → x402Middleware verifies signature + nonce
        → veniceClient.chat() → Venice AI API
```

### ERC-7710 Delegation Chain:
```
User (MetaMask: 0xdc6cBB97...) ─root delegation──► GekkoOrchestrator
GekkoOrchestrator ─sub-delegation──► GekkoResearcher  (spending caveat: $0.05 USDC)
GekkoOrchestrator ─sub-delegation──► GekkoValidator   (spending caveat: $0.03 USDC)
GekkoOrchestrator ─sub-delegation──► GekkoWriter      (spending caveat: $0.05 USDC)
```
Chain stored in `src/delegations.json`. Loaded at server start via `getDelegationChain()`.

### Real-time events (SSE):
```
Any agent action → event-bus.js → server.js rolling buffer (max 500)
  → GET /api/events/stream → page.tsx updates timeline, stepper, live feed
```

## Known Architecture Notes

- **Validator shares wallet with Researcher** — same private key, same address. Intentional (cost efficiency). Escrow skipped between them automatically.
- **1Shot relayer not used** — `src/oneshot.js` is a no-op stub. Payments go via `ethers.Wallet.sendTransaction()` on the signer. Falls back to `status:'simulated'` if wallet has no ETH for gas.
- **`DISPATCH_API_KEY` left blank** — `/api/goal` is open for local dev and frontend use.
- **`BUDGET.minBalance = 0`** — allows the demo to run with unfunded wallets.
- **Venice models**: use exactly `deepseek-v3.2`, `mistral-small-2603`, `llama-3.3-70b`. DO NOT use `venice-reasoning-preview` (404) or `mistral-31-24b` (404).
- **x402 search proxy**: strips `q`/`query` from body before forwarding to Venice (Venice rejects unknown keys).
- **ERC-7710 DelegationManager address** is `0x000...0` (placeholder) — MetaMask Delegation Toolkit v0.10.0 mainnet address not yet confirmed. Delegations are built and stored but `upgrade7702` is skipped in setup.
- **SSE event names**: `research_completed`, `validation_completed`, `synthesis_completed`, `goal_completed`, `payment_failed`, `escrow_created`, `escrow_released` are used by frontend stepper. Do not rename.
- **Circuit breaker**: in `src/circuit-breaker.js`, trips after 5 consecutive 5xx failures, resets after 30s. Wraps Venice client.
